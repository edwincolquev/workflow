# PROYECTO: PORTAL DE ANÁLISIS DE CADENA DE SUMINISTRO

## Contexto General

Somos una empresa importadora y distribuidora de repuestos automotrices con presencia nacional.

Actualmente utilizamos SAP Business One como ERP principal y SQL Server como base de datos operativa.

Durante los últimos años se desarrollaron diversos dashboards en Power BI para analizar diferentes áreas de la cadena de suministro.

Los dashboards existentes son:

1. Tránsitos
2. Productos Nuevos
3. Items Quebrados
4. Discontinuados
5. Inventarios (nuevo desarrollo)

Cada dashboard utiliza consultas SQL complejas directamente sobre SAP Business One y contiene reglas de negocio específicas desarrolladas y validadas por los usuarios.

El objetivo NO es reconstruir la lógica de negocio en Python.

El objetivo es construir una plataforma unificada que reutilice las consultas SQL existentes y permita evolucionar más rápidamente que Power BI.

---

# Objetivo Estratégico

Desarrollar un Portal de Análisis de Cadena de Suministro basado en Python y Streamlit que centralice todos los análisis operativos y gerenciales relacionados con:

* Inventarios
* Abastecimiento
* Importaciones
* Tránsitos
* Quiebres
* Productos nuevos
* Discontinuados
* Planificación futura

La plataforma debe convertirse en la herramienta principal de seguimiento para:

* Gerencia
* Compras
* Importaciones
* Logística
* Planificación de demanda

# Principios de Arquitectura

## Regla fundamental

La lógica de negocio debe permanecer en SQL Server.

Python NO debe recalcular reglas de negocio complejas que ya existen en SQL.

La arquitectura esperada es:

SAP Business One
→ SQL Server
→ Consultas SQL
→ Python / Pandas
→ Streamlit
→ Usuario Final
